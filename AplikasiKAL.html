Algoritma Pagerank#

Algoritma PageRank atau algoritma Google diperkenalkan oleh Lary Page, salah satu pendiri Google. Ini pertama kali digunakan untuk menentukan peringkat halaman web di mesin pencari Google. Saat ini, semakin banyak digunakan di berbagai bidang, misalnya dalam menentukan peringkat pengguna di media sosial dll… Yang menarik dari algoritma PageRank adalah bagaimana memulai dari masalah yang kompleks dan berakhir dengan solusi yang sangat sederhana. Dalam posting ini, saya akan mengajarkan Anda ide dan teori di balik algoritma PageRank. Anda hanya perlu memiliki beberapa dasar-dasar dalam aljabar dan Rantai Markov. Di sini, kita akan menggunakan peringkat halaman web sebagai kasus penggunaan untuk mengilustrasikan algoritma PageRank.

Algoritme PageRank adalah konsep dasar di balik cara Google Penelusuran memberi peringkat pada laman web. Ini pada dasarnya memberikan skor untuk setiap halaman web berdasarkan kepentingan dan relevansinya. Berikut rincian cara kerjanya:

Ide inti: Tautan sebagai suara

Bayangkan web sebagai jaringan raksasa halaman-halaman yang saling berhubungan, dengan link yang bertindak sebagai jembatan di antara halaman-halaman tersebut. PageRank menganggap tautan masuk ke laman web sebagai suara kepercayaan dan kepentingan. Semakin banyak link berkualitas tinggi yang diterima suatu halaman, semakin tinggi skor PageRank-nya.

Algoritma sedang beraksi

Pendekatan probabilistik: PageRank tidak hanya menghitung jumlah link. Ini menghitung kemungkinan pengguna mengklik link secara acak dan berakhir di halaman tertentu. Proses berulang: Algoritme dimulai dengan menetapkan skor PageRank awal (seringkali sama) untuk semua halaman web. Kemudian, secara berulang-ulang menghitung skor baru untuk setiap halaman berdasarkan skor PageRank dari halaman yang tertaut ke halaman tersebut. Bayangkan seorang pengguna mengklik halaman web secara acak; skor baru mempertimbangkan kemungkinan pengguna membuka halaman saat ini dari halaman web lain yang memiliki link. Faktor redaman: Untuk menghindari pantulan tanpa batas di sekitar link, algoritme menyertakan faktor redaman. Nilai ini, biasanya antara 0 dan 1, menunjukkan kemungkinan pengguna bosan dan menghentikan klik acaknya. Sisa kemungkinannya kemudian didistribusikan secara merata ke semua halaman web. Dampak dan evolusi

Meskipun PageRank adalah landasan kesuksesan awal Google, algoritma pemeringkatan mesin pencari telah berkembang jauh lebih canggih dari waktu ke waktu. Saat ini, ratusan faktor dipertimbangkan lebih dari sekadar tautan balik, termasuk kualitas konten, metrik keterlibatan pengguna, dan relevansi dengan kueri penelusuran.

Namun, prinsip inti PageRank – menggunakan tautan untuk menilai pentingnya laman web – tetap memberikan pengaruh yang signifikan terhadap cara Google memberi peringkat pada hasil penelusuran.

RUMUS DARI PERHITUNGAN PAGERANK

\[ r_j=\sum_{i \rightarrow j} \frac{r_i}{d_i} \]

Keterangan :

\(r_j\) = nilai dari node (halaman web) \(j\)

\(i\) = node (halaman web) yang menautkan (linking page)

\(j\) = node (halaman web) yang ditautkan (linked page)

\(r_i\) = nilai dari node (halaman web) \(i\)

\(i\) –> \(j\) = adanya link dari node \(i\) menuju node \(j\)

\(d_i\) = jumlah link keluar dari node \(i\) (derajat keluar)

\begin{aligned} & \left(r_0 \quad=\frac{r_4}{3}\right. \ & r_1 = \frac{r_2}{2}+\frac{r_4}{3}+r_3 \ & r_2=\frac{r_0}{3}+\frac{r_4}{3} \ & r_3=\frac{r_2}{2}+\frac{r_0}{3} \ & r_4=\frac{r_0}{3}+r_1) \ & \end{aligned}

import numpy as np

def kondisi(V0, V1):
    a = 0
    for i in range(len(V0)):  # Iterasi langsung pada nilai V0 dan V1
        a += (V1[i] - V0[i]) ** 2
    return a

def kali(mat, V0, V1):
    a = kondisi(V0, V1)
    print('a=',a)
    loop = 0
    while a < 0.05:  # Mengubah kondisi while
        loop += 1
        new = np.dot(mat, V1)  # Menggunakan np.dot untuk perkalian matriks
        a = kondisi(V1, new)
        print(new)
        print()
        V1 = new  # Update nilai V1
    print('loop=', loop)
